Guía de Restauración - Proyecto VitalScribe (v3.2)Este documento detalla cómo generar puntos de restauración y cómo recuperar el proyecto en caso de error crítico.1. Cómo crear un Punto de Restauración (Backup)Cada vez que alcances un hito estable, ejecuta el script de respaldo.Desde la Terminal de Visual Studio Code:Asegúrate de estar en la raíz del proyecto.Ejecuta el comando:.\scripts\create_restore_point.ps1
(Si PowerShell bloquea el script, ejecuta primero: Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass)Resultado:Se creará un Git Tag (ej. restore-point-v3.2-20231126-1200).Se creará un archivo .zip en la carpeta /backups.2. Método de Restauración (En caso de emergencia)Si una nueva funcionalidad rompe el proyecto, tienes dos formas de volver atrás.Opción A: Restauración vía Git (Recomendada)Este método revierte el código a un punto exacto en el tiempo sin perder el historial posterior (te mueve a un estado "detatched").Listar puntos disponibles:git tag
Restaurar al punto deseado:# Reemplaza con el nombre exacto del tag que quieres usar
git checkout restore-point-v3.2-YYYYMMDD-HHMM
Reinstalar dependencias limpias:npm ci
(Para volver al presente/última versión después de revisar, usa: git checkout main)Opción B: Restauración Física (Emergencia Total)Si Git está corrupto o prefieres empezar de cero desde el respaldo:Ve a la carpeta /backups.Descomprime el archivo restore-point-v3.2-....zip.Copia el contenido descomprimido y reemplaza los archivos en tu carpeta de proyecto.Ejecuta en la terminal:npm install
3. Protección contra SobrescrituraLos puntos de restauración generados usan git tag. Las etiquetas en Git son inmutables por defecto; no se pueden "sobrescribir" con un simple commit, lo que protege este estado del código contra errores accidentales de edición.
